
# super-hover

A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it stays tied to what's under your pointer while you scroll or when things move.

import { createSuperHover } from "super-hover";

const onEnter = (e: Event) => {

const onLeave = (e: Event) => {
    console.log("leave:", e.target);

root.addEventListener("superhoverenter", onEnter);
root.addEventListener("superhoverleave", onLeave);

## API

### `createSuperHover(options)`

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Returns a **cleanup** function that removes listeners and clears any active target.

### `UseSuperHoverOptions`

Shared options for **`super-hover/react`** (`useSuperHover`, `useSuperHoverRef`), **`super-hover/vue`** (`useSuperHover`), and **`super-hover/svelte`** (`superHover`). The root element is separate from these props: React passes the mounted root as the first argument to `useSuperHover`, or via the callback ref from `useSuperHoverRef`; Vue and Svelte attach to the element you bind `ref` or `use:superHover` to.

### Callbacks

Framework helpers register `onEnter` / `onLeave` on **`root`** for `enterEventType` / `leaveEventType`. Each handler receives the same bubbling DOM **`Event`** `createSuperHover` dispatches on the hit-tested row (a `CustomEvent` by default). **`event.target`** is that row element. Use `event.target as HTMLElement` (or narrower typing) when you read `dataset` or measure nodes.
