
A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it stays tied to what's under your pointer while you scroll or when things move.

## Usage

<CodeSnippet
  title="Basic usage"
  language="svelte"
  code={`
<script lang="ts">
  import { superHover } from "super-hover/svelte";

  const items = ["Alpha", "Beta", "Gamma"];
</script>

<ul class="space-y-1" use:superHover>
  {#each items as item (item)}
    <li
      data-super-hover
      class="rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100"
    >
      {item}
    </li>
  {/each}
</ul>
`}
/>

<ul

</ul>
`}

## API

### `createSuperHover(options)`

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Returns a **cleanup** function that removes listeners and clears any active target.

### Callbacks

The **`superHover`** action registers **`onEnter`** / **`onLeave`** on the host element for **`enterEventType`** / **`leaveEventType`**. Each handler receives the same bubbling DOM **`Event`** **`createSuperHover`** dispatches on the hit-tested row (a **`CustomEvent`** by default). **`event.target`** is that row element. Use **`event.target as HTMLElement`** (or narrower typing) when you read **`dataset`** or measure nodes.
