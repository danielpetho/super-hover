
A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it keeps tracking whatever sits under your pointer while you scroll or when things move on screen.

<Preview name="scroll-list" showCode={false} className="p-0! rounded-none! border-none!" description="Scrollable list comparing native vs super-hover; Scroll without moving the pointer." />

## Why use this?

Well, you probably don't need this. Browsers don't emit :hover while you're scrolling for performance reasons, which is obviously desired in most cases. But, by enforcing a fake hover state on every frame, it opens up the possibility of some interesting creative effects.

## Usage

Just simply import `superHover`, put `use:superHover` on the wrapper around your list (like the `ul` in the snippet), and mark each row with `data-super-hover`. Whatever is under the pointer picks up `data-super-hover-active`, so you can style “the hovered row” with classes such as Tailwind’s `data-[super-hover-active]:…`.

Out of the box it looks for `[data-super-hover]`, but you can customize the [selector](#api-create-super-hover-selector).

<CodeSnippet
  title="Basic usage"
  language="svelte"
  code={`
<script lang="ts">
  import { superHover } from "super-hover/svelte";

  const items = ["Alpha", "Beta", "Gamma"];
</script>

<ul class="space-y-1" use:superHover>
  {#each items as item (item)}
    <li
      data-super-hover
      class="rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100"
    >
      {item}
    </li>
  {/each}
</ul>
`}
/>

## Events

<Preview
  name="super-hover-events"
  showCode={false}

Pass `onEnter` and `onLeave` in the options object for `use:superHover` when you want logic tied to the active row: `onEnter` runs when hit-testing moves onto a new matched row (pointer moves or the list scrolls under a stationary cursor); `onLeave` runs on the row that stopped being active. Both handlers receive the event object, where `event.target` refers to that row element.

The preview above uses `onEnter` to swap the cover image so it matches the active row even while scrolling without moving the mouse.

<CodeSnippet
  title="Event handlers"
  language="svelte"
  code={`
<script lang="ts">
    import { superHover } from "super-hover/svelte";

    const items = ["Alpha", "Beta", "Gamma"];
</script>

<ul
    class="space-y-1"
    use:superHover={{
      onEnter: (e: Event) => {
        console.log("enter:", e.target);
      },
      onLeave: (e: Event) => {
        console.log("leave:", e.target);
      },
    }}
>
    {#each items as item (item)}
      <li data-super-hover class="rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100">{item}</li>
    {/each}
</ul>
`}
/>

## Props

<DocCodeHeading id="superhoveroptions">SuperHoverOptions</DocCodeHeading>

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Pass these fields to `createSuperHover(options)`. Returns a `cleanup` function that removes listeners and clears any active target.

Each frame the library reads `elementFromPoint`, walks ancestors with `closest(selector)` to pick one matched element, then optionally checks it lies inside `root`. `selector` defines which nodes may activate; `root` only limits where hits are considered, but it does not opt in every descendant.

<ApiPropList aria-label="SuperHoverOptions" anchorPrefix="create-super-hover">
  <ApiProp name="root" type="Document, Element, or omit" defaultValue={<em className="text-muted-foreground">whole document</em>}>
    Optional boundary: the matched element must lie inside this subtree; omit for the whole document. Does not make every descendant node a target; that is controlled by `selector` instead.
  </ApiProp>

  <ApiProp name="selector" type="string" defaultValue="[data-super-hover]">
    CSS selector passed to `element.closest` from the hit-tested node; defines which elements may activate (default `[data-super-hover]`). Independent of `root`, which only scopes where hits count.
  </ApiProp>

  <ApiProp name="activeAttribute" type="string" defaultValue="data-super-hover-active">
    Attribute toggled on the active matched element (empty string while active, removed otherwise). Use it for styling, e.g. `data-[super-hover-active]:…`.
  </ApiProp>

  <ApiProp name="enterEventType" type="string" defaultValue="superhoverenter">
    `CustomEvent` type dispatched on the matched element when it becomes active (`bubbles: true`).
  </ApiProp>

  <ApiProp name="leaveEventType" type="string" defaultValue="superhoverleave">
    `CustomEvent` type dispatched on the matched element when it stops being active.
  </ApiProp>
</ApiPropList>

<DocCodeHeading id="usesuperhoveroptions">UseSuperHoverOptions</DocCodeHeading>

Shape passed to the `superHover` action (`use:superHover`) from `super-hover/svelte` on the element that hosts the directive.

<ApiPropList aria-label="UseSuperHoverOptions" anchorPrefix="svelte-super-hover-action">
  <ApiProp name="enabled" type="boolean" defaultValue="true">
    When `false`, listeners are removed until enabled again.
  </ApiProp>

  <ApiProp name="selector" type="string" defaultValue="[data-super-hover]">
    CSS selector passed to `element.closest` from the hit-tested node; defines which elements may activate (default `[data-super-hover]`). Independent of `root`, which only scopes where hits count.
  </ApiProp>

  <ApiProp name="activeAttribute" type="string" defaultValue="data-super-hover-active">
    Attribute toggled on the active matched element (empty string while active, removed otherwise). Use it for styling, e.g. `data-[super-hover-active]:…`.
  </ApiProp>

  <ApiProp name="enterEventType" type="string" defaultValue="superhoverenter">
    `CustomEvent` type dispatched on the matched element when it becomes active (`bubbles: true`).
  </ApiProp>

  <ApiProp name="leaveEventType" type="string" defaultValue="superhoverleave">
    `CustomEvent` type dispatched on the matched element when it stops being active.
  </ApiProp>

  <ApiProp name="onEnter" type="(event: Event) => void" defaultValue="no-op">
    Runs when an enter event bubbles within `root`.
  </ApiProp>

  <ApiProp name="onLeave" type="(event: Event) => void" defaultValue="no-op">
    Runs when a leave event bubbles within `root`.
  </ApiProp>
</ApiPropList>

<DocCodeHeading id="callbacks">Callbacks</DocCodeHeading>

The `superHover` action registers `onEnter` / `onLeave` on the host element for `enterEventType` / `leaveEventType`. Handlers receive the same bubbling `Event` `createSuperHover` dispatches on the matched element (a `CustomEvent` by default). `event.target` is that element. Cast `event.target` when you read `dataset` or measure layout.
