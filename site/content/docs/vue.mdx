
A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it stays tied to what's under your pointer while you scroll or when things move.

## Usage

<CodeSnippet
  title="Basic usage"
  language="vue"
  code={`
<script setup lang="ts">
  import { useSuperHover } from "super-hover/vue";

  const items = ["Alpha", "Beta", "Gamma"];
  const rootRef = useSuperHover();
</script>

<template>
  <ul ref="rootRef" class="space-y-1">
    <li
      v-for="item in items"
      :key="item"
      data-super-hover
      class="rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100"
    >
      {{ item }}
    </li>
  </ul>
</template>
`}
/>

## API

### `createSuperHover(options)`

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Returns a `cleanup` function that removes listeners and clears any active target.

Each frame the library reads `elementFromPoint`, walks ancestors with `closest(selector)` to pick one matched element, then optionally checks it lies inside `root`. `selector` defines participation; `root` only limits where hits are considered—it does not opt in every descendant.

### Callbacks

`useSuperHover` registers `onEnter` / `onLeave` on the bound root element for `enterEventType` / `leaveEventType`. Handlers receive the same bubbling `Event` `createSuperHover` dispatches on the matched element (a `CustomEvent` by default). `event.target` is that element. Cast `event.target` when you read `dataset` or measure layout.
