
A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it keeps tracking whatever sits under your pointer while you scroll or when things move on screen.

## Usage

<CodeSnippet
  title="Basic usage"
  language="tsx"
  code={`
import { useSuperHoverRef } from "super-hover/react";

export function Example() {
  const rootRef = useSuperHoverRef();
  const items = ["Alpha", "Beta", "Gamma"];

  return (
    <ul ref={rootRef} className="space-y-1">
      {items.map((item) => (
        <li
          key={item}
          data-super-hover
          className="rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100"
        >
          {item}
        </li>
      ))}
    </ul>
  );
}
`}
/>

## Props

<DocCodeHeading id="superhoveroptions">SuperHoverOptions</DocCodeHeading>

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Pass these fields to `createSuperHover(options)`. Returns a `cleanup` function that removes listeners and clears any active target.

Each frame the library reads `elementFromPoint`, walks ancestors with `closest(selector)` to pick one matched element, then optionally checks it lies inside `root`. `selector` defines which nodes may activate; `root` only limits where hits are considered, but it does not opt in every descendant.

<ApiPropList aria-label="SuperHoverOptions" anchorPrefix="create-super-hover">
  <ApiProp name="root" type="Document, Element, or omit" defaultValue={<em className="text-muted-foreground">whole document</em>}>
    Optional boundary: the matched element must lie inside this subtree; omit for the whole document. Does not make every descendant node a target; that is controlled by `selector` instead.
  </ApiProp>

  <ApiProp name="selector" type="string" defaultValue="[data-super-hover]">
    CSS selector passed to `element.closest` from the hit-tested node; defines which elements may activate (default `[data-super-hover]`). Independent of `root`, which only scopes where hits count.
  </ApiProp>

  <ApiProp name="activeAttribute" type="string" defaultValue="data-super-hover-active">
    Attribute toggled on the active matched element (empty string while active, removed otherwise). Use it for styling, e.g. `data-[super-hover-active]:…`.
  </ApiProp>

  <ApiProp name="enterEventType" type="string" defaultValue="superhoverenter">
    `CustomEvent` type dispatched on the matched element when it becomes active (`bubbles: true`).
  </ApiProp>

  <ApiProp name="leaveEventType" type="string" defaultValue="superhoverleave">
    `CustomEvent` type dispatched on the matched element when it stops being active.
  </ApiProp>
</ApiPropList>

<DocCodeHeading id="usesuperhoveroptions">UseSuperHoverOptions</DocCodeHeading>

`UseSuperHoverOptions` for `useSuperHover(root, options)` and `useSuperHoverRef(options)` from `super-hover/react`. Pass the mounted root as the first argument to `useSuperHover`, or use the callback ref returned from `useSuperHoverRef`.

<ApiPropList aria-label="UseSuperHoverOptions (React)" anchorPrefix="react-use-super-hover">
  <ApiProp name="enabled" type="boolean" defaultValue="true">
    When `false`, listeners are removed until enabled again.
  </ApiProp>

  <ApiProp name="selector" type="string" defaultValue="[data-super-hover]">
    CSS selector passed to `element.closest` from the hit-tested node; defines which elements may activate (default `[data-super-hover]`). Independent of `root`, which only scopes where hits count.
  </ApiProp>

  <ApiProp name="activeAttribute" type="string" defaultValue="data-super-hover-active">
    Attribute toggled on the active matched element (empty string while active, removed otherwise). Use it for styling, e.g. `data-[super-hover-active]:…`.
  </ApiProp>

  <ApiProp name="enterEventType" type="string" defaultValue="superhoverenter">
    `CustomEvent` type dispatched on the matched element when it becomes active (`bubbles: true`).
  </ApiProp>

  <ApiProp name="leaveEventType" type="string" defaultValue="superhoverleave">
    `CustomEvent` type dispatched on the matched element when it stops being active.
  </ApiProp>

  <ApiProp name="onEnter" type="(event: Event) => void" defaultValue="no-op">
    Runs when an enter event bubbles within `root`.
  </ApiProp>

  <ApiProp name="onLeave" type="(event: Event) => void" defaultValue="no-op">
    Runs when a leave event bubbles within `root`.
  </ApiProp>
</ApiPropList>

<DocCodeHeading id="callbacks">Callbacks</DocCodeHeading>

`useSuperHover` and `useSuperHoverRef` register `onEnter` / `onLeave` on `root` for `enterEventType` / `leaveEventType`. Handlers receive the same bubbling `Event` `createSuperHover` dispatches on the matched element (a `CustomEvent` by default). `event.target` is that element. Cast `event.target` when you read `dataset` or measure layout.
