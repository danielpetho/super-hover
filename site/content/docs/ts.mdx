
A super tiny library that hit-tests hover every frame. Unlike native `:hover`, it stays tied to what's under your pointer while you scroll or when things move.

## Usage

<CodeSnippet
  title="Basic usage"
  language="typescript"
  code={`
import { createSuperHover } from "super-hover";

const list = document.querySelector("#list");
const superHover = createSuperHover({ root: list ?? undefined });

for (const el of document.querySelectorAll("#list li")) {
  el.setAttribute("data-super-hover", "");
  el.className =
    "rounded px-3 py-2 transition-colors data-[super-hover-active]:bg-neutral-100";
}

// later: superHover();
`}
/>

    console.log("enter:", e.target);
};
const onLeave = (e: Event) => {
    console.log("leave:", e.target);
};

root.addEventListener("superhoverenter", onEnter);

## API

### `createSuperHover(options)`

Registers pointer, scroll, and resize listeners and hit-tests on each animation frame. Returns a `cleanup` function that removes listeners and clears any active target.

Each frame the library reads `elementFromPoint`, walks ancestors with `closest(selector)` to pick one matched element, then optionally checks it lies inside `root`. `selector` defines participation; `root` only limits where hits are considered—it does not opt in every descendant.

### Callbacks

Listen on `root` for `superhoverenter` / `superhoverleave` (or your `enterEventType` / `leaveEventType`). `createSuperHover` emits bubbling `CustomEvent`s on the matched element; `event.target` points at that node. Remove listeners and call the `createSuperHover` cleanup when tearing down.
